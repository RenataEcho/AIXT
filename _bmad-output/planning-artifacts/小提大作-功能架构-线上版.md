## 0. 一句话定位

**小提大作** —— slogan「小提示，大作品」。
面向 **网文 / 短剧 / 漫剧 / 微电影** 的 **AI 内容创作 + 版权交易 + 收益分成** 一体化平台：把「小说 → 剧本 → 分镜 → 成片」的生产链条产品化，并叠加 **版权确权/交易 + 数据结算/提现 + 展厅/排行社区** 的商业闭环。

**架构主干**：
> **双创作入口，两条独立工作流**——「**写作**（小说/文本创作）」与「**自由画布**（漫剧视频制作）」× 资产/角色库 × 版权中心（登记授权）× 数据结算提现 × 光影展厅+排行社区，通过业务接口 `/api/fc/` 与 AI 引擎 `/api/synapse/` 双通道驱动，以虚拟货币 **米花/爆米花** 按次计费。

> ⚠️ 关键澄清（来自产品负责人）：
> - **写作**面向**小说文本**创作；**画布**面向**漫剧视频**制作。两者工作流程完全不同，不是同一链条的两种视图。
> - **变现现状 = 工作流付费**（画布内每次生成消耗米花/爆米花，走 `getCost` 计费）；**规划中**追加「版权分成」+「视频播放分成」（用户用工作流产出视频→产生播放收益→按比例反哺创作者）。
> - **版权中心 = 登记+授权申请**：版权均为**我方自有生产**或**合作方授权**，用户申请授权后方可制作漫剧视频，播放/付费分成目前走**线下分佣**。

## 3. 两条独立工作流（核心拆解）

产品有**两个入口、两套流程**，服务两类交付物。不要混为一谈。

### 3.1 流程 A —— 写作（小说/文本创作）

入口：`writing`（写作模式）　交付物：**小说/网文文本**
证据来自 `writing/editor` 与 `fine-outline` 代码。

```
作品（作品类型/写作风格/自定义要求）
  → 整体架构（大纲指令 → 大纲生成设置 → 生成大纲）
    → 细纲（细纲指令 → 章节细纲 → 新增/合并章节 → 章节排序）
      → 章节正文（章节生成设置 → 生成章节 → AI 正文 → 替换/重新生成）
        → 导出全文
```

- 关键能力：作品类型、写作风格、写作指令/大纲指令/细纲指令、自定义要求、关联素材、**前往角色库**（写作角色库）、章节目录管理、导出全文。
- 计费单位：**米花**（生成动作消耗）。

### 3.2 流程 B —— 自由画布（漫剧视频制作）

入口：`/canvas/:projectId?`（自由画布）　交付物：**漫剧/短剧视频**
画布是**节点图（node graph）+ 工作流（workflow）**双层结构，核心实体：`character/role 角色`、`scene 场景`、`shot 分镜`、`image 图`、`video 视频`、`model 模型`。

```
工作流 workflow（可创建/复用/复制，"工作流封面/我的工作流"）
  小说文本 →（小说转剧本/小说改剧本）→ 剧本（剧本模式）
    →（剧本转分镜）→ 分镜（分镜列表/分镜叙述/出镜角色/分镜选项）
      → 角色（生成角色图/角色库）+ 场景（场景调度台/场景库/生成场景图）
        →（生成分镜图/合成图片）→ 画面
          →（图生视频/文生视频/首尾帧生视频/精准动作控制器）→ 视频
            → 音频（上传音频/有无声音）+ 字幕擦除 + 姿势生成
              → 出视频结果 → 批量下载 / 下载全部分镜
```

- 节点分类：
  - **Agent/复合节点**：小说转剧本、剧本转分镜、全景图 agent、漫画故事 agent、场景调度台、多格电影分镜、动态视听推理。
  - **原子生成节点**：生成图片、生图节点、生成视频、图生视频、文生视频、首尾帧生视频、图片编辑器、姿势生成、字幕擦除、精准动作控制器。
  - **基础节点**：文本节点、图片节点、视频节点、音频传输、常规节点、自动运行节点（自动执行）。
  - **画布操作**：添加/整理/删除节点、平移画布、回到节点、复制节点 Id、面包屑导航。
- 计费单位：**爆米花**（每次生成走 `getCost`，可选生成数量/生成模式/使用模型）。

### 3.3 大作创建 masterpiece/create（漫剧项目入口）

- 内置**海量短剧爆款题材库**（如「一夜封神/上门女婿/丧尸围楼/义体改造」等数百标签，覆盖逆袭/战神/末世/赛博等赛道）。
- 含**主流程/主线任务**编排、**云端异步**生成、**下载全部分镜**、**交付**等能力 —— 更像"一键起项目 + 批量产出"的漫剧生产入口，下游衔接画布流程 B。
- 内置 21 语言（含俄/阿/印地/泰等），面向出海。

## 4. 技术侧观察（供架构参考）

- **前端**：Vue 3 + Element Plus，Vite 构建，路由懒加载分包（`vue-vendor` / `element-plus` / `utils` 三大 vendor + 各业务 chunk）。
- **双 API 通道**：
  - `/api/fc/` —— 业务接口（项目、版权、数据、账号等）。
  - `/api/synapse/` —— AI 引擎/Agent 执行（画布节点、生成、Agent 运行）。命名「synapse（突触）」暗示 AI 编排层。
- **画布关键接口**（实证）：`/api/chat/stream`（流式生成）、`/api/chat/getCost`（**计费/预估米花**）、`/api/chat/model/list`（模型列表）、`/api/chat/type/list`（能力类型）、`/api/chat/message/page`、`/api/chat/page`。→ 佐证「工作流付费 = 按次调用 AI 计费」。
- **鉴权**：多数工作台路由 `meta.requiresAuth:true`，游客仅可访问首页/展厅/文档/登录注册。
- **示例数据**：内置大量样例作品（《凡人修仙诀》《遮天传奇》《长安十二时辰》等 60+ 标题），覆盖玄幻/仙侠/都市/宫斗/科幻等题材，用于展厅/项目演示。
