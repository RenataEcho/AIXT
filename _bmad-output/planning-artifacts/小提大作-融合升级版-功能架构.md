## 0. 融合总原则（一句话）

> **保留小提大作的双创作入口（写作 / 画布）+ 版权中心 + 数据结算的骨架与样式，把大作联盟的「SOP 流水线、指挥舱、工具/提示词、我的空间」作为能力层灌进去，并在其上补齐「制作 → 售卖 → 分成 → 分佣」的一体化中台商业链路。**

三条硬约束：
1. **样式与 IA 不变**：沿用小提大作 Vue3 + Element Plus、左侧工作台导航、`/api/fc/` + `/api/synapse/` 双通道、米花/爆米花计费。
2. **不制造第二个产品**：大作联盟的功能一律**归位**到小提大作既有模块，能合并的合并，不能合并的作为该模块的「增强能力/子模式」。
3. **商业化优先补全链路**：制作、售卖、分成、分佣必须能在同一账户体系内跑通闭环。

## 1. 产品定位与目标

**小提大作（融合升级版）** —— 面向 **网文 / 剧本 / 短剧漫剧 / 微电影** 的
**「AI 创作 + 内容/工作流/知识库交易 + 版权合作 + 收益分成分佣」一体化中台**。

**产品目标（用户原话锚定）**：
> 集 **小说 / 剧本 / 视频 制作与售卖**、**工作流 / 知识库变现**、**版权合作**、**邀请分佣** 等商业链路于一体的中台产品。

**一句话架构**：
> 双创作入口（**写作**=小说/文本、**画布**=漫剧视频，画布内置大作联盟 SOP 流水线）
> × 资产/角色/提示词库 × 版权与文创中心（登记·授权·合作·确权）
> × 生产指挥舱（个人 SOP ↔ 团队 Gate 双视角）
> × 交易市场（作品 / 工作流 / 知识库 售卖·产权）
> × 结算中台（工作流付费 + 版权分成 + 播放分成 + 邀请分佣 → 提现 + 发票）
> 以米花/爆米花按次计费驱动，多语言出海。

## 4. 创作生产层详解（需求点①：大作联盟创作流程 → 小提大作画布）

产品维持**两个入口、两套流程**（不混为一谈），但**画布升级**为「自由节点图 + SOP 引导流水线」双模式。

### 4.1 流程 A —— 写作（小说/文本，保持不变）

```
作品(类型/风格/自定义要求) → 整体架构(大纲) → 细纲(章节细纲/排序) → 章节正文(AI生成/替换) → 导出全文
```
- 能力：写作角色库、关联素材、章节目录管理。计费：米花。
- 融合增益：可挂接「提示词库」（§6.1）与「版权确权」（写完可一键登记版权、挂牌售卖）。

### 4.2 流程 B —— 画布（漫剧视频）= 自由节点图 + SOP 引导模式【本次融合核心】

画布保留小提大作的**节点图（node graph）+ 工作流（workflow）**，并**内置大作联盟的 SOP 流水线**作为「引导模式」。

**双模式切换**：

| 模式 | 面向 | 形态 | 来源 |
|---|---|---|---|
| **自由画布** | 熟练用户 | 拖拽节点、连线、复用工作流 | 小提大作原生 |
| **SOP 引导模式** | 新手/标准化生产 | 逐阶段确认的流水线工作台 | 大作联盟创作流程 |

**SOP 引导阶段**（融合大作联盟流水线，落到画布）：
```
选题 topics → 风格 style → 剧本 script → 分镜 storyboard
  → 人物 characters → 场景 scenes → 画布合成 canvas → 图 image → 视频 video → 导出成片
```
- **严格流水线 gate**：前序未完成 gate 后序（`strictScriptAgent`，`completed` 逐项布尔）。
- **Agent 执行**：逐阶段产出落标准目录（`01_剧本/02_分镜/03_人物/04_场景/05_质检/06_风格`）。
- **状态机**：`idle/running`、`current_step`、`failed_step`、`retry`、`videoGenerated`、`canvasReady` —— 支持**失败重试 + 断点续跑**。
- **创作模式**：`film 微电影` / `serial-ip 连载IP`（`infinite 无限连载` / `microfilm`）。
- **风格/人物/场景增强**：555 风格·72 分类·IP 大师库、人物记忆一致性、场景锁定规则（「缺一项不得进视频」）。
- **两模式共享数据**：SOP 每阶段产物即画布节点，可随时切到自由画布继续精修。计费：爆米花（`getCost`）。

### 4.3 SOP 阶段 ↔ 画布节点映射

| SOP 阶段 | 画布节点（小提大作原生） |
|---|---|
| 选题/风格 | 小说文本节点 + 题材/风格参数 |
| 剧本 | 小说转剧本 / 剧本模式（Agent 节点） |
| 分镜 | 剧本转分镜（分镜列表/叙述/出镜角色） |
| 人物/场景 | 生成角色图 + 场景调度台 + 角色库/场景库 |
| 画布合成/图 | 生成分镜图/合成图片 |
| 视频 | 图生视频/文生视频/首尾帧/精准动作控制器 + 音频/字幕擦除 |
| 导出成片 | 批量下载 / 下载全部分镜 |

### 4.4 大作创建 masterpiece/create（保留，作为 SOP 快捷起点）
- 内置海量短剧爆款题材库 → 一键起项目 → 直接进入画布 SOP 引导模式。
- 与大作联盟「选题·爆款众筹」合并为同一题材入口。

## 5. 版权 / 文创中心（需求点②：两个产品文创功能融合并优化）

以小提大作版权中心为骨架，吸收大作联盟「IP 确权 + 封面对 + IP 大师/画廊」，并为商业化补「合作方入驻 + 产权」。

### 5.1 融合后的版权中心结构

| 子模块 | 路由（沿用小提大作） | 融合增强（来自大作联盟 + 商业化新增） |
|---|---|---|
| 版权首页/落地 | `/copyright` | 版权中心总入口 |
| 书籍列表/详情 | `copyright/books`、`book-detail` | 网文确权/交易 |
| 剧本详情 | `copyright/script-detail` | 剧本挂牌/交易 |
| 我的申请 | `copyright/my-apply` | 授权申请单/申请表（申请→审核→授权状态机） |
| 漫剧排行 | `ranking` | 排行榜 |
| **IP 确权** | `copyright/ip-verify`（新增） | 大作联盟 IP 确权 + 产权登记（作品/工作流/知识库通用） |

- **一套确权体系**覆盖：小说、剧本、漫剧视频、**工作流、知识库**（产权概念统一在此登记）。
- **授权链路状态机**：登记 → 挂牌/申请 → 审核 → 授权 → 制作/使用 → 分成结算（打通线上，替代原线下分佣）。
- **封面能力**与「提示词库」（§6.1）共享 IP 提示词模板，避免两套素材。

### 7.3 交易市场 market（新增左侧模块）

| 市场板块 | 售卖对象 | 计费/分成 |
|---|---|---|
| 作品市场 | 小说 / 剧本 / 漫剧成片 | 一次性买断 / 授权分成 |
| 工作流市场 | 可复用工作流（画布 workflow） | 售卖 / 订阅 / 按次授权 |
| 知识库市场 | 风格包 / 提示词包 / 素材·角色·场景库（**不含**教程/方法论类知识付费，决策⑤） | 售卖 / 订阅 |

### 7.4 结算中台（打通线上分成，替代线下分佣）
```
收入来源（付费/售卖/播放分成/版权分成/工作流·知识库变现/邀请分佣）
  → 数据看板 data/dashboard（收入·用量·播放）
    → 结算明细 data/settlement（分成规则·周期）
      → 提现 data/withdraw（【本期新建】）
      → 发票 data/invoice（【本期新建】：开票申请·抬头管理·开票记录·状态跟踪）
```

**本期重点新建（决策④）**：
- **提现功能**：结算余额 → 发起提现 → 审核 → 到账，含提现记录、状态、手续费/税费规则。
- **发票功能**：开票申请、发票抬头管理、开票记录、发票状态跟踪（对齐充值/消费/分佣的可开票金额）。

### 7.5 邀请分佣体系（沿用现有产品，决策④）
- **分佣层级与链路：现有产品已实现，本期不重做**（沿用既有邀请→绑定→返佣规则）。
- 佣金进入统一结算中台，与创作收益同账户，**本期补齐其「提现 + 发票」出口**（见 §7.4）。

### 7.6 播放分成数据来源（决策⑥）
- **用户自报**：粘贴抖音/视频号公开视频链接 → 审核 → 记录播放数据（沿用大作联盟播放链接奖励机制）。
- **第三方回传**：预留对接能力，**具体平台与回传协议待定**（见 §10）。
- 防作弊：绑定实名/设备/消费行为校验 + 自报数据人工/规则审核。
