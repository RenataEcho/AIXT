## 1. 产品定位

**大作联盟（AIXT）** —— 面向 **短剧 / 微剧 / 微电影** 的 **AI 内容生产平台**。

核心隐喻「生产指挥舱」：把 AI 创作全流程（选题 → 风格 → 剧本 → 分镜 → 人物/场景 → 成片）工业化、流水线化，并叠加 **积分成长 + 播放变现** 的运营闭环。

**一句话架构**：
> AI 创作流水线（选题→风格→剧本→分镜→人物/场景→画布成片） × 资源中心工具库 × 指挥舱调度 × 积分+播放变现增长飞轮，用多语言 + 多创作模式覆盖全球短剧/微电影生产。

## 6. 深挖：大赏（showcase）

> 证据来源：`23-export-archive-account-p02.js`（showcase 交互）、`28-showcase-comments-p01.js`（评论数据）、`27-showcase-playback-progress-p01.js`（播放进度）。

### 6.1 定位
大赏 = **作品展示广场 + 创作社区 + 从成片反向复盘创作流程的入口**。它是平台的「门面 + 增长入口」，把「好作品」「好评论」「可复盘的创作过程」聚合成一个可逛、可学、可互动的社区。

### 6.2 内容形态
- **手机墙（phone wall）**：多个竖屏「手机框」并排展示 AI 生成的短剧成片视频，每个手机框带**播放进度环**（`--showcase-progress`，随视频 timeupdate 实时更新），支持横屏/竖屏混排。
- **人群转盘（crowd turntable）**：一组人物角色围成环形，可转动/拖拽切换「C 位」角色，带聚焦放大（focus）、预加载、滚轮锁定等交互——强调「人物」而非按钮，让人脸居中。
- **作品卡信息**：标题、封面、作者、题材（现实/武侠/奇幻/历史等）。

### 6.3 社交与互动机制
- 五种互动：**点赞 / 收藏 / 评论 / 分享 / 关注**（`like/save/comment/share/follow`）。
- 其中 **点赞、收藏、评论、关注需登录**；未登录点击 → 弹登录浮层，登录后自动补执行原动作（`pendingShowcaseActionAfterLogin`）。分享无需登录。
- **评论系统**分三条泳道（lane）：
  - `hot` 热榜、`review` 评审、`debate` 争议。
  - 每条评论含：城市、作者、角色（观众/剪辑师/编剧/制片/产品经理…）、点赞数、情绪 `tone`（positive/mixed/critical）、标签（好评/建议/复盘/催更/共鸣/争议/镜头/字幕/文本…）。
  - 「我」发的评论初始 `role: 待审核`（评论需审核）。
- 首页可见「12 条新评论滚动」「2,846」等社区活跃度指标。

### 6.4 核心价值主张（从评论内容反推）
用户反复表达的诉求：**「能从成片倒看分镜/选题」「想看作者完整工程」「开放流程比开放成片更重要」**。→ 大赏的差异化不是炫成片，而是**「可复盘、可学习的创作过程」**，把成片作为流量入口，回流到创作流程模块。

### 6.5 与运营变现的关系
- 大赏作品的播放表现 ↔ **播放链接奖励**（抖音/视频号公开链接 → 审核 → 积分，阶梯 500/1万/5万/10万）。
- 社区互动（点赞/关注）→ 账号成长值/积分 → 套餐权益。
- **待确认**：大赏是否有正式「排行榜/评选」机制（评论中出现「排行榜」「催更」诉求，但当前更像展示墙+社区，评选规则尚未在代码中定型）。

## 7. 深挖：创作流程（两套并行的 SOP）

> 证据来源：`agent-run-data.js`（真实 Agent 试跑产物）、`production-sop-data.js`（指挥舱数据）、`06/07-runtime-profile`（strict-pipeline / 选题确认）。

### 7.1 关键发现：同一生产过程的「双视角」
平台把「一部短剧的生产」用两套视图表达：

| 视角 | 面向 | 模块 | 形态 |
|---|---|---|---|
| **创作 SOP（个人）** | 创作者本人 | 导航「创作流程」组 | 逐阶段工作台，边做边确认 |
| **指挥舱 G0–G9（团队）** | 制片/PMO/工厂 | `production-sop` | 看板/工厂调度，管人管进度管质检 |

### 7.2 创作 SOP（个人创作者流水线）
阶段顺序（来自 `agent-run-data` 的 `pipeline.completed` 与导航）：

```
选题 topic → 剧本 script → 风格 style → 分镜 storyboard
  → 人物 character → 场景 scene → 画布 canvas → 图像 image → 视频 video
```

- **strict-pipeline（严格流水线）**：`strictScriptAgent: true`，阶段有先后依赖，前序未完成会 gate 后序（`completed` 逐项布尔标记）。
- **agent-run（AI Agent 执行）**：由 AI Agent 逐阶段产出，产物落到标准目录：`01_剧本 / 02_分镜 / 03_人物 / 04_场景 / 06_风格 / 05_质检`。示例为「大作联盟21章正式试跑」，`plannedChapters: 21`（先确认目标章节数再跑）。
- **选题确认（topic-confirm）**：选题阶段用**单一 CTA 确认**（`topic-confirm-single-cta`），确认后才进入后续流水线。
- **画布（canvas）**：流水线末端的可视化编排台，把分镜图/视频素材放到无限画布上组装（素材面板、视频 dock、分组拖拽、连线），最终导出。
- 状态机字段：`status(idle/running)`、`current_step`、`failed_step`、`error_message`、`retry_button`、`videoGenerated`、`canvasReady` —— 支持失败重试与断点续跑。

### 7.3 指挥舱 SOP（G0–G9 团队调度）
生产被拆成 10 道「门（Gate）」，每道门有归属部门、任务数、待质检数、被打回数、平均停留天数、健康状态：

| Gate | 阶段 | 归属部门 | 状态示例 |
|---|---|---|---|
| G0 | 立项 | PMO | normal |
| G1 | 大纲 | 编剧中心 | attention |
| G2 | 剧本 | 编剧中心 | attention |
| G3 | 分镜 | 分镜导演 | risk |
| G4 | 资产 | 资产中心 | normal |
| G5 | 分镜图 | AI图像 | risk |
| G6 | 视频 | AI视频 | risk |
| G7 | 成片 | 剪辑声音 | normal |
| G8 | 审核 | QC合规 | attention |
| G9 | 复盘 | PMO | normal |

指挥舱额外管理的对象：
- **员工/部门/角色**（编剧中心、分镜导演、资产中心、AI图像、AI视频、剪辑声音、QC合规、PMO），带人均产能 capacity。
- **项目**（P001「雨棚名单」…P008，各自处于某个 Gate）。
- **日报**：完成内容、产出数、工具（如 Premiere）、**废图率 / 废视频率**（generated/usable/waste）、返工次数与原因、blocker、求助对象、明日计划。
- **质检（QC）**：提交人、交付链接、结果、原因编码。
- **返工记录**、**原因编码**。
- **资产库**：角色（锁定规则/参考图/年龄/身份/造型）、场景（空间类型/年代/光线/道具/锁定规则）——「缺一项不得进视频」。
- **提示词库**：按阶段/类型分类，记录使用次数、成功率、负责人、锁定状态。

### 7.4 两套流水线的映射关系（推断）
| 创作 SOP（个人） | ≈ 指挥舱 Gate（团队） |
|---|---|
| 选题 topic | G0 立项 + G1 大纲 |
| 剧本 script | G2 剧本 |
| 分镜 storyboard | G3 分镜 |
| 人物/场景 characters/scenes | G4 资产 |
| 图像 image / 画布 canvas | G5 分镜图 |
| 视频 video | G6 视频 |
| （导出成片） | G7 成片 |
| — | G8 审核（QC） |
| （大赏复盘） | G9 复盘 |

> **待确认**：个人创作 SOP 与指挥舱是否共享同一份数据（即个人产出是否自动进入 Gate 看板），还是两套独立数据源。这是判断产品是「个人工具」还是「团队工厂 OS」的关键。
